Hi Kakul, I thought I'd share with you why GitHub is meaningful to me and what I think I could bring to the product team...

I like GitHub a lot, and have used it for a lot of things

Product and process

Aside from being able to browse the world's biggest collection of open-source software, version control allows users to follow its development. Opening GitHub is like walking into a constantly-updating, global library that I can contribute to.

Social coding

I see GitHub's primary goal as helping people work together. GitHub is code, but ultimately you're innovating and removing friction from collaboration. I think the solutions to the challenges we face as a planet will come from this kind of open team workflow.

For all

I'm not sure Apple would've made Swift open-source if it wasn't for GitHub. And Java certainly wouldn't be as ubiquitous. And I wouldn't be using Bootstrap for this site. GitHub helps make open-source the default. This availability of these tools democratizes software development and innovation - a big win.

I know your product

I've used GitHub for issue tracking, to manage engineers, to manage my own code, to code collaboratively. I've used it to find and learn new libraries, to pick a d3 layout to use, to see what my friends are building.

I bring experience and curiosity

I have the experience of building a funded, 17-person company (starting with just me and a laptop). I have the experience of managing a remote team of engineers, of motivating a team after a pivot, of exciting investors about a product vision and of dissolving a company.

I feel as comfortable positioning products as I do writing AJAX post requests as I do designing mockups and measuring CACs and MAUs. I want to build meaningful products and am curious about anything that can help me do that better.
